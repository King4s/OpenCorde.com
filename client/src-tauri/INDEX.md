# /client/src-tauri/

Tauri 2.0 desktop application shell for OpenCorde.

## Purpose

Wraps the SvelteKit frontend in Tauri to create distributable desktop apps (.exe/.dmg/.AppImage) with:

- System tray integration
- Native OS notifications
- Auto-launch on startup
- Deep-link protocol handling (opencorde://)
- Secure credential storage (OS keychain)
- Settings persistence

## Structure

### Configuration

- `Cargo.toml` — Rust dependencies and metadata
- `build.rs` — Tauri build script
- `tauri.conf.json` — Tauri app config (windows, tray, bundle settings)
- `capabilities/default.json` — Tauri 2.0 capabilities (permission model)

### Rust Code

- `src/main.rs` — Thin entry point
- `src/lib.rs` — App builder, plugin registration, deep-link handling, tray setup
- `src/tray.rs` — System tray icon and context menu
- `src/notifications.rs` — Desktop notification utilities
- `src/mobile.rs` — Mobile-specific init (push permission request for iOS/Android, `#[cfg(mobile)]`)
- `src/commands/` — IPC command handlers (see commands/INDEX.md)

### Assets

- `icons/` — Platform-specific icons (generated by `pnpm tauri icon`)

## Tauri 2.0 Features

1. **System Tray**
   - Left-click shows/focuses main window
   - Menu: "Open OpenCorde", "Check for Updates", "Quit"
   - Close main window → hides to tray (not quit)

2. **Deep Links**
   - Register `opencorde://` protocol
   - Handled by `on_uri_scheme_request` in lib.rs
   - Forwarded to frontend via "deep-link" event

3. **Plugins**
   - `tauri-plugin-shell` — Execute system commands
   - `tauri-plugin-notification` — Native desktop notifications
   - `tauri-plugin-autostart` — Auto-launch from settings
   - `tauri-plugin-deep-link` — URI scheme registration
   - `tauri-plugin-os` — Platform detection

4. **Mobile Targets (iOS + Android)**
   - `tauri.conf.json` → `bundle.iOS` and `bundle.android` sections added
   - iOS: `bundle.iOS.developmentTeam` must be set for App Store signing
   - Android: `bundle.android.minSdkVersion = 24` (Android 7.0+)
   - Mobile init: `src/mobile.rs` — requests push permission on first launch
   - See `../MOBILE_BUILD.md` for full build instructions

5. **Credential Storage**
   - `keyring` crate for OS-level credential storage
   - Windows: Credential Manager
   - macOS: Keychain
   - Linux: Secret Service (libsecret)

## Development

### Build

```bash
cd client
pnpm install
pnpm tauri dev
```

### Generate Icons

```bash
pnpm tauri icon path/to/icon.png
```

### Package for Distribution

```bash
pnpm tauri build
```

Outputs to `src-tauri/target/release/bundle/` with platform-specific formats.

## Integration with SvelteKit

- Frontend builds to `build/` via `adapter-static`
- Tauri serves static files from `build/` in production
- Dev mode: serves from `http://localhost:5173` (vite dev server)
- IPC commands invoked via `@tauri-apps/api` (already in package.json)

## Logging

- Structured logging via `tracing` crate
- Level: INFO by default
- Override with `RUST_LOG` env var (e.g. `RUST_LOG=debug pnpm tauri dev`)
